Module&Npm发布

一、模块的分类：

  1. 文件模块 require('./a.js') 自定义模块

  2. 原生模块 核心模块fs http path vm ... 不需要安装 require(fs)

  3. 第三方模块 ma（ 设置缩进） bluebird（promise） 先去安装，当前目录下的node_modules 文件夹

  • 三方模块
    默认第三方模块会去当前的node_module目录下查找，会找名字相同的文件，默认找index.js；(如果当前node_module没有这个模块 他会像上查找这个目录（向上找node_modules这个文件夹）

console.log(module.paths);

let file = require('mz');

  • 自定义模块
    如果自定义的模块 ,新版本会先查找文件 文件没有找目录，下面的index.js

// 如果自定义的模块 

// 新版本会先查找文件 文件没有找目录

require('./a/index.js');

二、Npm

npm可以帮我们安装模块（安装node 会自动安装npm）

模块的安装分为两类

  1. 全局安装（在命令行中安装）-g

  2. 本地安装（在代码中使用）

命令的使用：

  1. --save：开发上线都需要；在package.json中dependencies

  2. --save-dev\-D：只有开发中使用；在package.json中devDependencies

管理工具：
npm install yarn -g
npm install nrm -g（可以切换源）

  • npm ls： 列出所有源

  • npm use taobao：使用淘宝源

nvm：切换node版本

  • nvm win：（未测试是否可以更改node版本）

发布：全局的包
目的：在本地，把我的项目变成一个全局的命令，

  1. npm init -y

  2. 在package.json中

      "bin": {

        "js-pack": "./bin/js-pack"

      },

  3. 建文件夹bin->js-pack文件,内容如下：

    

    #! /usr/bin/env node  //表示在node环境下执行

    

    console.log('呵呵');

  4. 把包链接到npm下npm link

  5. cmd执行：js-pack打印”呵呵”

发布到npm上

  1. npm addUser

  2. 输入用户名、密码

  3. npm publish(注意：这里要使用npm源)

  4. npm install js-bj-pack -g：安装自己的包；ok；成功啦啦啦啦

注意：需要在本地，先把她打包出es5，在发布到npm上
